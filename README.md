# Ad Generator

An interactive web application for creating AI-generated advertising banners. Built on Nuxt 4 + Cloudflare Workers, using the same infrastructure stack as [Backtick](https://backtick.cogitations.com).

Hosted at: **adgen.cogitations.com**

---

## How it works

Each ad is composed of three AI-generated image layers, assembled by a layout engine:

1. **Hero image** — A photorealistic scene generated by Fal.ai (Flux 2 Pro) from a free-text prompt you write.
2. **Background** — A flat solid color generated by Fal.ai from a color description (e.g. "dark navy").
3. **Bullet list** — A vertical timeline graphic generated by Fal.ai from your step definitions (icon + label pairs).

All three images are sent to **Imejis.io** along with the text content (headline, subheadline, body, CTA). Imejis composes them onto a fixed design template and returns the final JPEG.

The result is stored in **Cloudflare R2** and displayed in the browser.

## Database schema

### `ad_configs`
Stores the parameters for a reusable ad configuration.

| Column | Type | Description |
|--------|------|-------------|
| `id` | integer PK | Auto-increment |
| `name` | text | Display name for the config |
| `headline` | text | Main headline text |
| `subheadline` | text | Subheadline text |
| `body_text` | text | Body copy paragraph |
| `cta_text` | text | CTA button label |
| `hero_image_prompt` | text | Fal.ai prompt for the hero photo |
| `background_description` | text | Color description for the background (e.g. "dark navy") |
| `bullet_steps` | text | JSON array of `{icon: string, label: string}` |
| `created_at` | integer | Unix timestamp |
| `updated_at` | integer | Unix timestamp |

### `generated_ads`
Tracks each generation run for a config.

| Column | Type | Description |
|--------|------|-------------|
| `id` | integer PK | Auto-increment |
| `ad_config_id` | integer FK | References `ad_configs.id` |
| `status` | text | `pending` \| `generating` \| `complete` \| `error` |
| `r2_key` | text | Key of the JPEG in the `adgen-images` R2 bucket |
| `error_message` | text | Error detail if status is `error` |
| `created_at` | integer | Unix timestamp |

npx wrangler secret put NUXT_IMEJIS_DESIGN_ID

## Local development

### First-time setup

1. Install dependencies:
   ```bash
   npm install
   ```

2. Copy and fill in secrets:
   ```bash
   cp .dev.vars.example .dev.vars
   # Edit .dev.vars with your API keys
   ```

3. Start the dev server (migrates DB, builds, launches Wrangler):
   ```bash
   npm run dev
   ```
   Open **http://localhost:8787**

### Subsequent runs

```bash
npm run dev
```

The `dev` script is: `npm run db:migrate && npm run build && cp .dev.vars .output/.dev.vars && npx wrangler dev`

Local D1 state lives in `.wrangler/` and persists across rebuilds.

2. Set production secrets (if not already set):
   ```bash
   npx wrangler secret put NUXT_FAL_KEY
   npx wrangler secret put NUXT_IMEJIS_API_KEY

   ```

3. Deploy:
   ```bash
   npm run deploy
   ```

### Subsequent deploys

```bash
npm run deploy
```

If the database schema changed, also run `npm run db:migrate:remote` before or after deploying.

The custom domain `adgen.cogitations.com` is configured in `wrangler.toml` and must be set up in the Cloudflare dashboard (Workers & Pages → Custom Domains).

## Adding a database migration

1. Modify the schema in [server/utils/db/schema.ts](server/utils/db/schema.ts).

2. Generate the SQL migration:
   ```bash
   npm run db:generate
   ```
   This creates a new `.sql` file in `drizzle/migrations/`.

3. Apply locally and verify:
   ```bash
   npm run db:migrate
   ```

4. Apply to production before or after deploying:
   ```bash
   npm run db:migrate:remote
   ```

The migration runner (`scripts/db-migrate.mjs`) is idempotent — it tracks applied migrations in a `__drizzle_migrations` table and skips files that have already run.

## Image generation pipeline

The `generate.post.ts` API route runs this pipeline synchronously:

```
User clicks "Generate Ad"
        │
        ▼
POST /api/ad-configs/[id]/generate
        │
        ├─ Create generated_ads record (status: generating)
        │
        ├─ Fal.ai (3 requests in parallel):
        │   ├─ Hero image  ← from heroImagePrompt field
        │   ├─ Background  ← from backgroundDescription field
        │   └─ Bullet list ← from bulletSteps array
        │
        ├─ Imejis.io render API
        │   └─ Composites all 3 images + text → JPEG binary
        │
        ├─ Upload JPEG to R2 (adgen-images bucket)
        │
        └─ Update generated_ads record (status: complete, r2_key: ...)
                │
                ▼
        Browser renders /api/images/[r2Key]
```

Fal.ai requests use the queue API with polling (3-second intervals, up to 60 attempts). The entire pipeline typically takes 30–60 seconds. Cloudflare Workers on a paid plan support up to 15-minute wall-clock time; free tier is limited to 30 seconds — use a paid plan for production.

## Fal.ai model

All three images use `fal-ai/flux-2-pro`. The model is called via Fal's queue REST API directly (no SDK dependency). To switch models, update the model string in [server/api/ad-configs/[id]/generate.post.ts](server/api/ad-configs/%5Bid%5D/generate.post.ts).
